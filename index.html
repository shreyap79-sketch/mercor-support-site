Q1: Using the Chrome Dev Tools, find the `interviewId` and `status` of your Talent Success Interview

Answer

  1. Open the Interviews page and press F12 to open DevTools.
  2. Go to Network, refresh once, and filter for interview (or view Fetch/XHR requests).
  3. Open the request that returns the interview list (e.g., /api/interviews).
  4. In the Response tab (JSON), read: 
    {
  "id": "INTERVIEW_ID",         // interviewId
  "type": "Talent Success Interview",
  "status": "Submitted"         // status as shown on the UI badge
}
  5. The status here is Submitted; the JSON also provides the unique interviewId.
Q2: When you update your work experiences on the Resume page, what internal endpoint is called and with which HTTP Method? Is this the right HTTP method to use, why or why not?

Answer

  1. On the Resume page, edit a Work Experience item with DevTools ‚Üí Network open, then click Save.
  2. The request is typically to an endpoint like /api/profile/experiences/:id.
  3. The method used for editing is commonly PATCH (correct for partial updates). PUT is acceptable if the entire object is replaced; POST is for creating a new record.
  4. Conclusion: PATCH is the right REST choice here because only selected fields are being updated.

Q1: Write a SQL query to return a list of users who applied to job listings created by the company named "Mercor".

SELECT DISTINCT
  u.UserId,
  u.Email
FROM Applications a
JOIN Listings    l ON l.ListingUID = a.ListingVersionId
JOIN Company     c ON c.CompanyId  = l.CompanyId
JOIN MercorUsers u ON u.UserId     = a.UserId
WHERE c.Name = 'Mercor';

Q2: Write a SQL query to return the average minimum rate and average maximum rate (in dollars) of all job listings created by the company named "Mercor".

SELECT
  AVG(l.RateMin) / 100.0 AS AvgMinRateUSD,
  AVG(l.RateMax) / 100.0 AS AvgMaxRateUSD
FROM Listings l
JOIN Company  c ON c.CompanyId = l.CompanyId
WHERE c.Name = 'Mercor';

Note: rates are stored in cents; divide by 100 for USD.
